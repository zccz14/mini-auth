Deployment notes

Static hosting stays first-class

Use relative local assets for this page, then configure mini-auth to trust whatever final page origin you deploy.

  • Publish the contents of demo/ so index.html, ./style.css, and ./main.js stay path-safe on GitHub Pages.
  • Set mini-auth --origin to the final browser page origin, then use ?sdk-origin= when the docs page and auth server are on different origins.
  • Custom GitHub Pages domains should ship a matching CNAME file so the browser origin stays stable for WebAuthn and CORS.

Known issues

Current caveats, not permanent contracts

Call out the real issues that affect integration, but avoid turning temporary behavior into a product promise.

  • Passkeys need a valid RP ID and a compatible browser context.
  • Cross-origin pages must be listed in --origin.
  • Multi-tab refresh races are a current SDK bug.
