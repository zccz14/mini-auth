mini-auth

A small, self-hosted auth server for apps that just need auth.

For teams that want email OTP, passkeys, JWTs, JWKS, and SQLite without adopting a larger platform.

    Deployment notes
    
          
    Static hosting stays first-class
    
          
    
            Use relative local assets for this page, then configure mini-auth to
            trust whatever final page origin you deploy.
          
    
        
    

        
    
          
      
            
    • GitHub Pages project paths should still load ./style.css.
      • 
            
    • Custom domains should keep a stable browser origin for passkeys.
      • 
            
    • The auth server still needs the page origin in --origin.
      • 
          
    
        
    
      
    

      
    
        
    
          
    Known issues
    
          
    Current caveats, not permanent contracts
    
          
    
            Call out the real issues that affect integration, but avoid turning
            temporary behavior into a product promise.
          
    
        
    

        
    
          
      
            
    • Passkeys need a valid RP ID and a compatible browser context.
      • 
            
    • Cross-origin pages must be listed in --origin.
      • 
            
    • Multi-tab refresh races are a current SDK bug.
